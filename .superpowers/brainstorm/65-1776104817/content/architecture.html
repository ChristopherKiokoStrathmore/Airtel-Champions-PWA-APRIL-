Security model
🔐

Password hashing

pgcrypto crypt(password, gen_salt('bf')) — bcrypt with cost factor 10. Never stored in plaintext. Verified server-side via Supabase RPC.

🔒

Row Level Security

RLS on all 4 tables. TLs can only read/write their own rows. The promoter MSISDN unique constraint is enforced at the DB level — not just in the app.

🧱

Locked reports

Once is_locked = true, an RLS policy blocks any UPDATE or DELETE on promoter_gas_entries for that report. Immutable at DB level.

🚫

MSISDN uniqueness

UNIQUE constraint on promoter_members.msisdn across all Team Leads. The DB rejects duplicate promoter assignments — no app-layer check needed.
